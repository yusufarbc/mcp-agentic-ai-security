# Contributing to MCP Agentic AI Security Review

Thank you for your interest in contributing to the MCP Agentic AI Security Review repository! We welcome contributions from the community to help advance the understanding of security in Agentic AI systems.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please read it to understand the expected behavior and positive environment we aim to foster.

## How to Contribute

There are several ways you can contribute:

### 1. Reporting Issues
*   **Bugs:** If you find errors in the documentation, code, or website, please [open an issue](https://github.com/yusufarbc/mcp-agentic-ai-security/issues) describing the problem.
*   **Suggestions:** Have an idea for a new section, a research paper to include, or an improvement? Open an issue to discuss it!

### 2. Submitting Content
*   **Research Papers:** If you have found a relevant academic paper, please add it to the `reference/` directory and update `reference/Readme.md` with a summary.
*   **Documentation:** Improvements to the `docs/` files are always welcome. Ensure your writing style matches the existing academic/professional tone.

### 3. Improving the Website
*   The website is built with HTML/CSS in the `website/` directory.
*   Feel free to submit Pull Requests (PRs) for design improvements, responsiveness fixes, or accessibility enhancements.

## Pull Request Process

1.  **Fork the repository** to your own GitHub account.
2.  **Create a branch** for your feature or fix (`git checkout -b feature/amazing-feature`).
3.  **Commit your changes** (`git commit -m 'Add amazing feature'`).
4.  **Push to the branch** (`git push origin feature/amazing-feature`).
5.  **Open a Pull Request** against the `main` branch of this repository.

## Style Guide

*   **Language:** All content must be in **English**.
*   **Tone:** Professional, academic, and objective.
*   **Formatting:** Use standard Markdown. Ensure all links are valid.
